单目录下makefile文件

  1. 在/home/xxt目录下创建makefile-226目录，在此目录下创建源代码目录src1,在src1目录下创建六个源代码文件，分别是input.c、compute.c、test.c、input.h、compute.h和main.h，其依赖关系如图9所示。

  • 如果没有安装make使用sudo apt install make来安装

  1. Makefile原型：
    在makefile-226目录下复制src1为src2目录, 在src2目录下创建makefile文件, 通过该makefile文件编译生成目标文件*.o，再连接成可执行文件test。运行可执行文件，观察执行结果；

  2. 引入用户变量：
    在makefile-226目录下复制src2为src3目录, 在src3目录下创建makefile文件, 以步骤（2）为基础，创建引入用户自定义变量的makefile文件, 通过该makefile文件编译生成目标文件*.o，再连接成可执行文件test。运行可执行文件，观察执行结果；

  3. 引入自动变量：
    在makefile-226目录下复制src3为src4目录,以步骤（3）为基础，在src4目录下创建引入自动变量的makefile文件, 通过该makefile文件编译生成目标文件*.o，再连接成可执行文件test。运行可执行文件，观察执行结果；

  4. 引入模式规则：
    在makefile-226目录下复制src4为src5目录,, 以步骤（4）为基础，在src5目录下创建引入模式规则的makefile文件, 通过该makefile文件编译生成目标文件*.o，再连接成可执行文件test。运行可执行文件，观察执行结果；

  5. 引入隐规则：
    在makefile-226目录下复制src5为src6目录,, 以步骤（5）为基础，在src6目录下创建引入隐规则的makefile文件, 通过该makefile文件编译生成目标文件*.o，再连接成可执行文件test。运行可执行文件，观察执行结果。

伪目标

  1. 利用伪目标进行打包压缩和删除操作：
    在makefile-226目录下复制src6为src7目录,, 以步骤（一）（6）为基础，在src7目录下创建引入伪目标的makefile文件。 在当前目录下执行命令：make可以编译生成目标文件*.o，再连接成可执行文test,并将当前目录所有文件打包压缩成test-112.tar.gz。执行命令：make clean可删除所有目标文件*.o和可执行文件test.


  2. 利用伪目标进行安装和卸载操作：
    在makefile-226目录下复制src7为src8目录,, 以步骤(二)（1）的Makefile为基础，增加可安装和卸载功能。 在当前目录下执行命令：make可以编译生成目标文件*.o，再连接成可执行文test,并将当前目录所有文件打包压缩成test-112.tar.gz。执行命令：make clean可删除所有目标文件*.o和可执行文件test。执行命令：make install可将可执行文件test复制到/usr/bin目录下。执行命令：make uninstall可将test文件从/usr/bin目录删除。
